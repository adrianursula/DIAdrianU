Versión 0.14: Metadatos ID3 y Editor de Notas (Diario)

3 de noviembre de 2025

Hitos Clave:

  • Lectura de Metadatos (ID3): Se integró la librería jsmediatags para leer metadatos de archivos MP3. Al reproducir una pista, la app ahora intenta leer sus tags ID3 (título, artista) y los muestra en el reproductor (#current-track-info) en lugar del nombre del archivo.
  • Nuevo Diálogo de Notas: Se añadió un nuevo icono (edit_note) a la cabecera del calendario que abre un diálogo modal para escribir notas sobre el día actualmente seleccionado (focusedDate).
  • Persistencia de Notas (IndexedDB): Se creó un nuevo "Store" (Notes) en IndexedDB. Las notas de texto se guardan automáticamente (con "debounce") a medida que el usuario escribe, asociadas a una clave de fecha (YYYY-MM-DD).
  • Indicador de Notas: Los días que tienen una nota guardada ahora muestran un pequeño indicador visual (.has-note::before) en la esquina superior, similar al indicador de audio.

Notas o Retos:

La integración de jsmediatags fue sencilla para la lectura de tags básicos. La lógica de guardado de notas requirió implementar un "debounce" para no escribir en IndexedDB en cada pulsación de tecla, optimizando el rendimiento. Esta versión sienta las bases funcionales completas del "Diario Sónico".

Versión 0.11: Memoria de Reproducción y Selector de Tema

31 de octubre de 2025

Hitos Clave:

  • Memoria de Reproducción (IndexedDB): La aplicación ahora guarda la última pista y el segundo exacto (currentTime) en IndexedDB al pausar o cada 3 segundos. Al recargar la app (tryLoadSavedFolder), restaura automáticamente la pista y el tiempo donde se dejó.
  • Diálogo de Ajustes: Añadido un nuevo icono de "Ajustes" (settings) en la cabecera que abre un diálogo modal.
  • Selector de Modo (Claro/Oscuro/Sistema): Dentro de Ajustes, el usuario puede forzar el modo claro (light), modo oscuro (dark) o respetar la configuración del sistema (system). La preferencia se guarda en IndexedDB.
  • Selector de Color de Acento: El usuario puede elegir entre múltiples temas de color (ej. Terracota, Azul, Verde). La paleta seleccionada se aplica dinámicamente y se guarda en IndexedDB.

Notas o Retos:

Esta fue una gran actualización de IndexedDB. Se creó un nuevo store ("Playback") para la memoria de reproducción y se expandió el store "Settings" para el tema. La lógica de loadPlaybackPosition fue compleja, ya que debía buscar el archivo por nombre en todas las listas de fechas de audioFiles.

Versión 0.6: Reproducción Continua, Botón "Hoy" y Regex Flexible

26 de octubre de 2025

Hitos Clave:

  • Reproducción Continua (Cola de Reproducción): Al seleccionar un audio desde el diálogo de múltiples archivos, la aplicación ahora crea una "cola de reproducción". Al finalizar un audio, (ended event), se reproduce automáticamente el siguiente de la lista para ese día.
  • Botón "Hoy" (Navegación Rápida): Se añadió un botón "Hoy" (icono today) en la cabecera del calendario. Al pulsarlo, la vista salta inmediatamente al mes actual y selecciona (hace clic) en el día de hoy.
  • Flexibilidad en Nomenclatura (Regex Mejorado): Se actualizó la expresión regular (Regex) para detectar fechas. Ahora busca el patrón YYYY-MM-DD en *cualquier parte* del nombre del archivo (.match(/(\d{4}-\d{2}-\d{2})/)), en lugar de exigir que esté solo al principio.

Notas o Retos:

Esta versión se centra enteramente en la calidad de vida y la robustez. La reproducción continua era la pieza que faltaba para la gestión de múltiples audios. El botón "Hoy" es un estándar de UX en cualquier calendario. Finalmente, el Regex flexible hace que la aplicación sea 10 veces más útil en un escenario real, donde los usuarios no nombran sus archivos de forma tan estricta.

Prompt 2: Generación de la Aplicación "Audio Calendario" (V1)

Actúa como un programador senior y diseñador de aplicaciones multiplataforma, experto en UI/UX y en la implementación de sistemas de diseño modernos. Tu especialidad es crear interfaces limpias, funcionales y accesibles.
Nuestro Proyecto:
Vamos a construir, paso a paso, una aplicación web que funciona como un reproductor de audios locales integrado con un calendario. El objetivo es que un usuario pueda ver un calendario, hacer clic en un día específico y que la aplicación reproduzca un archivo de audio asociado a esa fecha.
Pila Tecnológica (Stack):

HTML5 (para la estructura semántica)
CSS3 (para los estilos)
JavaScript (ES6+) (para toda la lógica y la interactividad)
Requisito Fundamental de Diseño:
Toda la aplicación, desde la estructura HTML hasta el último detalle de CSS, debe basarse estricta e íntegramente en el sistema de diseño Material Design 3 (MD3). Debes usar los principios de MD3 para los colores (paletas dinámicas, tokens), la tipografía, los componentes (botones, cards, superficies) y la iconografía (usando Material Symbols).
Requisitos Funcionales Clave:

Calendario Interactivo: La interfaz principal mostrará un calendario (mes actual). El usuario debe poder navegar entre meses.
Selección de Día: El usuario puede hacer clic en cualquier día del calendario.
Reproducción de Audio: Al hacer clic en un día, la aplicación debe buscar un archivo de audio local correspondiente a esa fecha (p. ej., 2025-10-21.mp3) y reproducirlo.
Manejo de Archivos Locales: Dado que JavaScript en el navegador tiene restricciones de seguridad para acceder a archivos locales directamente, deberás proponer la mejor arquitectura para esto. Para la Versión 1, podemos empezar usando un elemento <input type="file" webkitdirectory directory multiple> para que el usuario seleccione la carpeta donde almacena sus audios. La aplicación luego mapeará esos archivos a las fechas del calendario.
Nuestro Proceso de Trabajo (Iterativo):
Seguiremos un modelo de desarrollo ágil.

Versión 1 (Producto Mínimo Viable):
Comienza proporcionando el código completo (HTML, CSS y JavaScript) para la Versión 1.
Esta versión debe incluir:
