Bitácora de Desarrollo

Historia de la construcción, retos técnicos y soluciones aplicadas.

Fase 1: Cimientos

Inicialización y Backend

Se inició el proyecto con Expo SDK 52 y se integró Supabase como backend as a service.

SUCCESS: Conexión a Supabase establecida.
FEATURE: Auth Context creado para manejar sesiones de usuario y persistencia.
CHALLENGE: Definir esquemas RLS para asegurar que los nuevos usuarios solo vean sus propios datos.
Fase 2: Estilado y UI

Reto NativeWind

Implementación de un diseño "Premium" oscuro. Nos encontramos con un problema crítico al configurar NativeWind/Tailwind.

ERROR: @tailwind directives not recognized.
FIX: Se crearon manualmente `postcss.config.js` y `tailwind.config.js` y se forzó la reinstalación de `postcss` y `autoprefixer`.

"El diseño no es solo cómo se ve, sino cómo funciona. NativeWind nos permitió iterar rápido."

Fase 3: Visualización

Gráficos Interactivos

Creación del componente DonutChart. Hubo dificultades con la librería de SVG en Expo Go.

ERROR: RNSVG module not found.
SOLUTION: Reinstalación limpia de `react-native-svg` y rebuild del prebuild local.
RESULT: Gráficos animados y reactivos al filtrar gastos vs ingresos.
Fase 4: Multi-Idioma

Internacionalización (i18n)

Implementación in-house de un sistema de traducción mediante React Context.

  • Creación de diccionarios JSON para ES y EN.
  • Bug Crítico: El cambio de idioma no se reflejaba en toda la app.
  • Solución: Se descubrió que `LanguageProvider` no envolvía correctamente a `_layout.tsx` raíz. Se corrigió y envolvió toda la navegación.
Fase 5: Entrega

APK y Documentación

Generación del ejecutable Android usando EAS Build en la nube y creación de esta web.
